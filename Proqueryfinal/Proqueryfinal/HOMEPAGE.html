SQL INTRO

What is SQL?

SQL, or Structured Query Language, is a powerful language used to communicate with databases. It allows you to perform various operations on data, such as:

  1. Creating new databases and tables
  2. Retrieving data from tables
  3. Updating existing data
  4. Deleting unwanted data



What can SQL do?

  1. SQL allows you to run queries against a database.
  2. SQL is capable of retrieving data from a database.
  3. SQL can insert data in a database.
  4. SQL can be used to update database records.
  5. SQL is capable of deleting records from a database.
  6. SQL has the ability to create new databases.
  7. SQL is capable of creating new tables in a database.
  8. SQL can produce stored procedures in a database.
  9. SQL has the capability to create views in a database.
  10. SQL has the ability to set permissions on tables, procedures, and views.



RDBMS

  1. RDBMS stands for Relational Database Management System.
  2. RDBMS is the basis for SQL, and for all modern database systems such as MS SQL Server, IBM DB2, Oracle, MySQL, and Microsoft Access.
  3. RDBMS stores data in database objects known as tables. A table is a collection of related data entries organized in columns and rows.



Key Concepts:

  1. Tables: A collection of related data organized into rows and columns.
  2. Rows: Individual records within a table. A record is a horizontal entity in a table.
  3. Columns: Specific fields within a table.
  4. Primary Key: A unique identifier for each row in a table.
  5. Foreign Key: A column that references the primary key of another table, establishing a relationship.
  6. Joins:Combining rows from two or more tables based on a related column.

SQL CREATE TABLE

The CREATE TABLE keyword is used to create a new table in a database.

SYNTAX

CREATE TABLE table_name (
   column1 datatype,
   column2 datatype,
   .... );

The column parameters specify the names of the columns of the table. The datatype parameter specifies the type of data the column can hold (e.g. varchar, integer, date, etc.).



CREATE TABLE EXAMPLE

EXAMPLE

CREATE TABLE Employees (
   EmployeeID int,
   LastName varchar(255),
   FirstName varchar(255),
   Address varchar(255),
   City varchar(255) 
);

The PersonID column is of type int and will hold an integer.

The LastName, FirstName, Address, and City columns are of type varchar and will hold characters, and the maximum length for these fields is 255 characters.



The empty "Employees" table will now look like this:

SQL ALTER TABLE

The ALTER TABLE keyword is used to add, delete, or modify columns in an existing table. Is also used to add and drop various constraints on an existing table.




ALTER TABLE - ADD COLUMN

To add a column in a table, use the following syntax:

SYNTAX

ALTER TABLE tableName;
ADD column_name datatype;


The following SQL adds an "Email" column to the "Employees" table:

EXAMPLE

ALTER TABLE Employees;
ADD Email varchar(255);


ALTER TABLE - DROP COLUMN

To delete a column in a table, use the following syntax (notice that some database systems don't allow deleting a column):

SYNTAX

ALTER TABLE table_Name;
DROP COLUMN column_name;

The following SQL deletes the "Email" column to the "Customers" table:

EXAMPLE

ALTER TABLE Employees;
DROP COLUMN Email;


ALTER TABLE - RENAME COLUMN

To rename a column in a table, use the following syntax:

SYNTAX

ALTER TABLE table_Name;
RENAME COLUMN old_name to new_name;

To rename a column in a table in SQL Server, use the following syntax:

SYNTAX

EXEC sp_rename 'table_name.old_name' 'new_name, 'COLUMN';



ALTER TABLE - ALTER/MODIFY DATATYPE

To change the data type of a column in a table, use the following syntax:

SYNTAX

ALTER TABLE table_name;
ALTER COLUMN column_name datatype;

SQL INSERT INTO

The INSERT INTO keyword is used to insert new records in a table.


It is possible to write the INSERT INTO statement in two ways:

Step 1. Specify both the column names and the values to be inserted:

SYNTAX

Return all columns from the Employees table: 

INSERT INTO table_name(column1, column2, column3, ...);
VALUES (value1, value2, value3, ...);


Step 2. If you are adding values to all of the table's columns, there is no need to specify the column names in the SQL query. Make sure the values are in the same order as the table's columns. Here, the INSERT INTO syntax would be the following:

SYNTAX

INSERT INTO table_name;
VALUES (value1, value2, value3, ...);

SQL DELETE

The DELETE keyword is used to delete existing records in a table.

SYNTAX

   DELETE FROM table_Name;
   WHERE condition;

Note: When updating records in a database, always exercise caution. If the WHERE clause is omitted in an UPDATE statement, all records in the table will be updated, which may lead to unintended data changes. Similarly, when deleting records, pay close attention to the WHERE clause in the DELETE statement. The WHERE clause specifies which record(s) to delete. Omitting it will result in the deletion of all records in the table, which can have critical consequences.

SYNTAX

Note: The WHERE clause is utilized in UPDATE, DELETE, and other statements in addition to SELECT statements!

SQL AGGREGATE FUNCTIONS

An aggregate function is a function that performs a calculation on a set of values, and returns a single value.


Aggregate functions are often used with the GROUP BY clause of the SELECT statement. The GROUP BY clause splits the result-set into groups of values and the aggregate function can be used to return a single value for each group.



These are the SQL aggregate functions that are most frequently used:

  1. MIN- returns the smallest value within the selected column
  2. MAX - returns the largest value within the selected column
  3. COUNT- returns the number of rows in a set
  4. SUM- returns the total sum of a numerical column
  5. AVG- returns the average value of a numerical column



The SQL MIN() Functions


The MIN() function returns the smallest value of the selected column.

EXAMPLE

Understand the syntax of MIN in query

   SELECT MIN(column_name)
   FROM table_Name;
   WHERE condition;



The SQL MAX() Functions


The MAX() function returns the smallest value of the selected column.

EXAMPLE

Understand the syntax of MAX in query

   SELECT column_name()
   FROM table_Name;
   WHERE condition;



The SQL COUNT() FUNCTION

The COUNT() function returns the number of rows that matches a specified criterion.

EXAMPLE

Understand the syntax of COUNT in query

   SELECT COUNT column_name()
   FROM table_Name;
   WHERE condition;



The SQL SUM() FUNCTION

The SUM() function returns the total sum of a numeric column.

EXAMPLE

Understand the syntax of SUM in query

   SELECT SUM column_name()
   FROM table_Name;
   WHERE condition;



The SQL AVG() FUNCTION

The AVG() function returns the average value of a numeric column.

EXAMPLE

Understand the syntax of AVG in query

   SELECT AVG column_name()
   FROM table_Name;
   WHERE condition;

SQL INNER JOIN

`

Records with matching values in both tables are selected using the INNER JOIN keyword.

SYNTAX

SELECT column_name(s)
FROM table1
INNER JOIN table2
ON table1.column_name = table2.column_name;


Let's look at a selection of the Products table:

ProductID ProductName CategoryID Price
1 NESTEA 01 20
2 TANG 02 21
3 ETC 03 22


And a selection of the Categories table:

CategoryID CategoryName Description
01 Beverages Soft drinks, coffees, teas, beers, and ales
02 Condiments Savory sauces, spreads, and seasonings
03 Confections Desserts, candies, and sweet breads

We will join the Products table with the Categories table, by using the CategoryID field from both tables:

EXAMPLES

Join Products and Categories with the INNER JOIN keyword:


SELECT ProductID, ProductName, CategoryName
FROM  Products
INNER JOIN Categories
ON Products.CategoryID = Categories.CategoryID;

Note: Only rows that match in both tables are returned by the INNER JOIN keyword. This implies that a record would not be returned in the result if you had a product with no CategoryID or one whose CategoryID was missing from the Categories table.



NAMING THE COLUMNS

It is a good practice to include the table name when specifying columns in the SQL statement.


EXAMPLES

Join Products and Categories with the INNER JOIN keyword:


SELECT Products.ProductID, Products.ProductName, Categories.CategoryName
FROM Products
INNER JOIN Categories
ON Products.CategoryID = Categories.CategoryID;

OR

EXAMPLES

Join Products and Categories with the INNER JOIN keyword:


   SELECT p.ProductID, p.ProductName, c.CategoryName 
   FROM Products p
   INNER JOIN Categories c
   ON p.CategoryID = c.CategoryID;

SQL GROUP BY


The GROUP BY statement creates summary rows like "find the number of customers in each country" by grouping rows with the same values.


To group the result-set by one or more columns, the GROUP BY statement is frequently used when combined with aggregate functions (COUNT(), MAX(), MIN(), SUM(), and AVG()).

EXAMPLES

SQL SUBQUERIES


SQL queries that are nested inside one another are called subqueries. Multiple queries can be contained inside a subquery, one after the other. They help us to choose particular rows at runtime that meet specific requirements. The query that contains them is called the outer query or outer select, and they are also referred to as the inner query or inner select.

  1. Subqueries are nested inside statements like SELECT, INSERT, UPDATE, DELETE, or any other subquery.

  2. Subqueries are present in the WHERE clause, FROM clause, or HAVING clause of the PARENT SQL query.

  3. They are used with comparison operators and logical operators lik >, <,>=, <>, <=, SOME, ANY, ALL, and IN.

  4. They execute before the outer query at the run time and pass the result to complete the statement.

  5. Subqueries are used to compare an expression to the output and check if any row gets selected.

EXAMPLES

(SELECT [DISTINCT] subquery_select_parameter
FROM {table_name | view_name} 
         {table_name | view_name} ...
[WHERE  search_conditions]
[GROUP BY column_name [,column_name ] ...]
[HAVING search_conditions] )

SINGLE ROW SUBQUERY

Single-row subqueries are subqueries that give their parent query a single row as an output. A SQL SELECT statement with a FROM clause and a comparison operator, a WHERE clause, or a HAVING clause uses single-row subqueries. The SELECT statement makes use of single-row subqueries. Let's look at an example.

EXAMPLE

SELECT name AS  painting, 
   price
(SELECT AVG) (price)
FROM paintings) AS avg_price
FROM paintings;

The above SQL query displays the average price of all paintings besides their original price. The result looks like this:


Painting Price avg_Price
Patterns 5000 5700
Gift 3200 5700
Curiosity 8900 5700


MULTIPLE ROW SUBQUERY

Multiple-row subqueries are subqueries that give their parent query an output of several rows. A SQL SELECT statement that has a HAVING clause, WHERE clause, FROM clause, and a logical operator (ALL, IN, NOT IN, and ANY) can use multiple row subqueries. Let's examine it with an example.


EXAMPLE

SELECT AVG (agency_fee) 
FROM sales_agents
WHEREid NOT IN (SELECT id
               FROM managers);


The average agency fee for all sales agents who are not managers is determined by the SQL query above. The list of all managers' IDs is returned by the subquery. The outer query then determines the average agency charge for the sales agents who are not managers by filtering the table to discover their records. It gives back a single average agency charge value. The following is how the output appears:


agency_fee
1885


CORRELATED SUBQUERIES

Correlated subqueries are subqueries that, based on the data retrieved from the parent query, deliver many columns as output. It is difficult to understand since the inner and outside queries are interdependent.

WHERE and FROM clauses in SELECT statements allow for the use of linked subqueries. Let's use an example to better comprehend it.

EXAMPLE

SELECT city
(SELECT COUNT (*)
FROM paintings b
WHERE g.id = p.gallery_id) total_paintings
FROM galleries g;

The quantity of paintings in each gallery is determined by the SQL query above. For the total number of paintings in the associated query, the subquery returns a single value. Each painting's details, along with the city of the art gallery where it is displayed, are returned by the outer query. The following's result is:

NESTED SUBQUERIES

Subqueries that are inside another subquery are called nested subqueries. Subqueries are executed level by level. The innermost is executed first, and then the outer ones. Let's see some examples.

EXAMPLE

SELECT name AS painting,
         price,
(SELECT AVG (price)
FROM paintings 
WHERE price in
(SELECT price
FROM paintings

AS avg_Price
FROM paintings;

SQL AGGREGATE FUNCTIONS

An aggregate function is a function that performs a calculation on a set of values, and returns a single value.


Aggregate functions are often used with the GROUP BY clause of the SELECT statement. The GROUP BY clause splits the result-set into groups of values and the aggregate function can be used to return a single value for each group.



These are the SQL aggregate functions that are most frequently used:

  1. MIN()- returns the smallest value within the selected column
  2. MAX() - returns the largest value within the selected column
  3. COUNT()- returns the number of rows in a set
  4. SUM()- returns the total sum of a numerical column
  5. AVG()- returns the average value of a numerical column



The SQL MIN() Functions


The MIN() function returns the smallest value of the selected column.

EXAMPLE

   FROM table_Name;
   WHERE condition;

The MAX() function returns the smallest value of the selected column.
